2.2 Hexo 配置和命令

Hexo中文文档-配置，文档很全，很详细，想要用好Hexo，还是去看文档吧；下面的内容适用于你初次使用。

2.2.1 配置

打开_config.yml ，下面是一个hexo网站的配置；

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
keywords 网站的关键词。使用半角逗号 , 分隔多个关键词。
author 您的名字
language 网站使用的语言
timezone 网站时区。Hexo 默认使用您电脑的时区。时区列表。比如说：America/New_York, Japan, 和 UTC 。

2.2.2 常用的命令

命令 格式 说明
init hexo init [folder] 新建一个网站
generate hexo generate 或者hexo g 生成静态文件
server hexo server 或者 hexo s 启动服务器
clean hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)
new hexo new [layout] 新建一篇文章，layout 是可选参数
deploy hexo deploy 部署hexo到远处仓库，比如GitHub

2.3 部署到GitHub

在 “2.1 一分钟启动你的博客”中，我们只是在本地启动了博客，如果想要让别人也看到你的网站，就需要把它部署到一个所有人都能看到的地方，GitHub Pages 是一个不错的选择；GitHub 帮我们省去了繁琐的建站流程和建站的必要花费（购买服务器，IP，域名等）。

2.3.1 创建GitHub Pages

  • 打开GitHub账号，导航到 New Repository , 创建一个Repository Name为“youraccountname.github.io”的公有仓库，这个名字特殊的仓库，就是GitHub Pages；

2.3.1 将Hexo博客部署到GitHub Pages

  • 打开_config.yml,找到对应的位置，用创建的GitHub Pages的仓库地址替换“repo”位置的地址；

    注意：”branch” 必须是master，GitHub Pages 默认使用master 分支；

  • 使用hexo deploy 命令将本地代码推送到GitHub Pages

  • 在浏览器地址栏输入https://youraccountname.github.io
