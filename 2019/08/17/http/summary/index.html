Posted on 2019-08-17 | In HTTP拾遗 |

1 URI 与URL

1.1 URI

URI全称Uniform Resource Identifer，中文名为统一资源标识符，URI就像因特网上的邮政地址一样，在世界范围内唯一标识并定位信息资源。

如http://n.sinaimg.cn/ent/w630h354/20190803/963d-iatixpk7456112.jpg 标识了一个Web服务器上一个图片资源的位置。

URI分为两种形式，URL和URN

1.2 URL

URL全称 Uniform Resource Locator, 中文名为统一资源定位符，它是URI的一种常见形式;

格式为：<scheme>://<user>:<password>@<host>:<port>/<path>;<params>?<query>#<frag>

组件 描述
scheme 访问服务器获取资源时要使用的协议
user 某些协议访问资源时需要用户名，如ftp://anonymous@ftp.prep.ai.mit,edu/pub/gnu
password 用户名后面可能要包含密码
host 服务器的主机名或者IP地址
port 服务器监听的端口，HTTP默认为80,HTTPS默认为443
path 服务器上资源的路径，由一个/开始
params 某些协议会指定输入参数，可以包含多个参数字段，相互之间用;隔开,如ftp://ftp.prep.ai.mit,edu/pub/gnu;type=d
query 某些协议要传递查询参数，以?开始，参数(key=value)之间以&分割，如http://www.joes-hardware.com/inventory-check.cgi?item=12731&color=blue
frag 引用部分资源或者一个片段，一个片段资源的名称，只在客户端内部使用以#开始

4 浏览器请求远程资源的过程

  • 浏览器从URL中解析出服务器的主机名
  • 浏览器将服务器的主机名转换成服务器的IP地址
  • 浏览器将端口（如果有的话）从URL中解析出来
  • 浏览器建立一条与Web服务器的TCP连接
  • 浏览器向服务器发送一条HTTP请求报文
  • 服务器向浏览器回送一条HTTP响应报文
  • 关闭连接，浏览器显示文档
